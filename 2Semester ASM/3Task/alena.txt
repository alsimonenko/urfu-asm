Тебе в 3task можно сослаться на то, что я библиотеку выложил, а можно и аккуратно распарсить допустимые аргументы.
Примерно так:
cmd_line = "_(1)_(2)[_(3)...]", т.е.
    Ровно один пробел,
    (1) - одно из чисел 0,1,2,3,4,5,6,7,D,E,F,10
    Опять пробел,
    (2) - одно из чисел 0,1,2,3,4,5,6,7

    далее может быть пробел, если он есть, значит есть и третий аргумент.
    Находить его значение не нужно, поэтому надо просто определить есть ли этот пробел.

Если все эти случаи аккуратно на джампах рассмотреть, то можно и не подключать библиотеку вовсе.
Единственный момэнт - (1), который может быть 10.

Но можно поступить так:
 I. Если cmd_len < 4 (_X_Y), ошибка.

 II. Пусть ab=(1), то есть a=cmd_line[1], b=cmd_line[2]
 Если (('0' <= a <= '7') || ('D' <= a <= 'F')) && (b == ' '), OK бери char_to_int из библиотеки, вот тебе cmd_arg1
 Если (a == '1') && (b == '0'),                           OK cmd_arg = 10h
 Иначе ошибка

 III. Пусть a=(2), то есть a=cmd_line[3 или 4, в зависимости от предыдущего]
 Если ('0' <= a <= '7'), OK cmd_arg2 = a - '0' = a - 48
 Иначе ошибка

 IV. Теперь, если cmd_len > 4 или 5,  зависимости от результатов II, есть еще аргументы, считаем, исполняем ввод символа
